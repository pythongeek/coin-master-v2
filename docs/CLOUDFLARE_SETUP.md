# 🛡️ Cloudflare DDoS প্রোটেকশন সেটআপ গাইড

আপনার সার্ভার ও ইউজারদের সুরক্ষিত রাখতে Cloudflare ব্যবহার করুন। এটি **সম্পূর্ণ বিনামূল্যে**।

## কেন Cloudflare দরকার?

ক্রিপ্টো বেটিং সাইট প্রায়ই DDoS আক্রমণের শিকার হয় (প্রতিযোগী বা দুষ্টু ইউজাররা সাইট ডাউন করতে চেষ্টা করে)। Cloudflare এই আক্রমণ আপনার সার্ভার পর্যন্ত পৌঁছানোর আগেই আটকে দেয়।

## ধাপ ১: Cloudflare অ্যাকাউন্ট তৈরি করুন

1. [cloudflare.com](https://cloudflare.com) এ যান এবং ফ্রি অ্যাকাউন্ট তৈরি করুন
2. "Add a Site" চাপুন এবং আপনার ডোমেইন লিখুন
3. **Free Plan** বেছে নিন

## ধাপ ২: DNS রেকর্ড সেটআপ

Cloudflare আপনার বর্তমান DNS রেকর্ড স্ক্যান করবে। নিশ্চিত করুন:

| টাইপ | নাম | মান | প্রক্সি স্ট্যাটাস |
|------|-----|-----|-------------------|
| A | @ | আপনার VPS IP | 🟠 Proxied (চালু) |
| A | www | আপনার VPS IP | 🟠 Proxied (চালু) |

**গুরুত্বপূর্ণ:** "Proxied" (কমলা মেঘ আইকন) অবশ্যই চালু রাখবেন — এটিই DDoS প্রোটেকশন দেয়।

## ধাপ ৩: নেমসার্ভার পরিবর্তন

Cloudflare আপনাকে দুটি নেমসার্ভার দেবে (যেমন `aria.ns.cloudflare.com`)। আপনার ডোমেইন রেজিস্ট্রার (Namecheap, GoDaddy ইত্যাদি) এ গিয়ে নেমসার্ভার পরিবর্তন করুন। DNS প্রোপাগেশনে ২৪ ঘণ্টা পর্যন্ত সময় লাগতে পারে।

## ধাপ ৪: SSL/TLS সেটিং

Cloudflare ড্যাশবোর্ডে **SSL/TLS** ট্যাবে যান:

- Encryption mode: **Full (strict)** বেছে নিন (আপনার সার্ভারে Let's Encrypt সার্টিফিকেট থাকার পর)
- **Always Use HTTPS**: চালু করুন
- **Automatic HTTPS Rewrites**: চালু করুন

## ধাপ ৫: নিরাপত্তা সেটিং (সবচেয়ে গুরুত্বপূর্ণ)

**Security** ট্যাবে যান:

- **Security Level**: Medium বা High করুন
- **Bot Fight Mode**: চালু করুন (ফ্রি, বট আক্রমণ আটকায়)
- **Challenge Passage**: 30 মিনিট

## ধাপ ৬: রেট লিমিটিং রুল (Free Plan-এ সীমিত)

**Security → WAF → Rate limiting rules** এ গিয়ে নতুন রুল যোগ করুন:

```
যদি: URI Path contains "/api/auth/login"
এবং: একই IP থেকে ১ মিনিটে ৫ বারের বেশি রিকোয়েস্ট
তাহলে: Block (১০ মিনিটের জন্য)
```

এটি ব্রুট ফোর্স লগইন আক্রমণ প্রতিরোধ করবে।

## ধাপ ৭: Page Rules (ঐচ্ছিক, পারফরম্যান্স বাড়াতে)

**Rules → Page Rules** এ:

```
URL: yourdomain.com/api/*
সেটিং: Cache Level = Bypass (API রেসপন্স কখনো ক্যাশ হবে না)
```

## যাচাই করুন

সব ঠিকভাবে কাজ করছে কিনা চেক করতে:

```bash
# DNS প্রোপাগেট হয়েছে কিনা চেক করুন
nslookup yourdomain.com

# Cloudflare এর মাধ্যমে ট্রাফিক যাচ্ছে কিনা চেক করুন (Server header দেখুন)
curl -I https://yourdomain.com
# "Server: cloudflare" দেখতে পাবেন
```

## গুরুত্বপূর্ণ সতর্কতা

আপনার Nginx config-এ Cloudflare-এর IP থেকে আসা রিকোয়েস্ট চিনতে `X-Forwarded-For` হেডার সঠিকভাবে পড়তে হবে — এটি ইতিমধ্যে `nginx.prod.conf`-এ সেটআপ করা আছে।

সরাসরি আপনার VPS IP-তে কেউ অ্যাক্সেস করার চেষ্টা করলে (Cloudflare বাইপাস করে), সেটি ব্লক করতে UFW ফায়ারওয়ালে শুধু Cloudflare-এর IP রেঞ্জ থেকে ৮০/৪৪৩ পোর্টে অ্যাক্সেস দিতে পারেন (অপশনাল, অ্যাডভান্সড সেটআপ — তালিকা: https://www.cloudflare.com/ips/)।
