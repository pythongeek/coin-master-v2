## দুর্বলতা রিপোর্ট করা

CryptoFlip আর্থিক লেনদেন ও Provably Fair সিস্টেম নিয়ে কাজ করে, তাই নিরাপত্তা আমাদের কাছে সর্বোচ্চ অগ্রাধিকার।

নিচের যেকোনো বিষয়ে দুর্বলতা পেলে **পাবলিক GitHub Issue খুলবেন না**:

- Provably Fair অ্যালগরিদমে ম্যানিপুলেশনের সুযোগ
- ব্যালেন্স/পেআউট হিসাবে কারচুপির সুযোগ
- অথেন্টিকেশন বাইপাস
- SQL ইনজেকশন বা XSS
- ওয়ালেট সিগনেচার যাচাইয়ে দুর্বলতা

পরিবর্তে, সরাসরি রিপোজিটরি মেইনটেইনারের কাছে প্রাইভেট মেসেজ পাঠান বিস্তারিত বর্ণনাসহ:

- দুর্বলতার বিবরণ
- পুনরুৎপাদনের ধাপ
- সম্ভাব্য প্রভাব

আমরা ৪৮ ঘণ্টার মধ্যে প্রতিক্রিয়া জানানোর চেষ্টা করব।

## নিরাপত্তা সেরা চর্চা (যারা এই কোড ডেপ্লয় করবেন তাদের জন্য)

- `.env` ফাইল কখনো গিট-এ কমিট করবেন না
- `JWT_SECRET` ও `ADMIN_SECRET_KEY` শক্তিশালী র‍্যান্ডম ভ্যালু দিয়ে সেট করুন
- প্রোডাকশনে অবশ্যই HTTPS ব্যবহার করুন (Let's Encrypt সেটআপ আছে)
- নিয়মিত নির্ভরতা (dependencies) আপডেট করুন: `npm audit`
- ডাটাবেস ব্যাকআপ নিয়মিত নিন এবং সুরক্ষিত স্থানে রাখুন
- সার্ভার সিড রোটেশন নিয়মিত করুন (এডমিন প্যানেল থেকে)
